# Lore Characters

This folder holds per-character canon and character-specific visual reference for `LORE`.

Each major character should have a dedicated folder.

## Standard character folder layout

Each character folder should usually contain:
- `character.md`: the main reference canon for that character
- `summary.md`: the prose-facing read on that character
- `concept-art/`: grouped visual reference for that character when relevant

## What belongs here

- identity and role in the story
- public perception versus hidden truth
- relationships
- locked reveals tied to that character
- character-specific unresolved questions
- character-specific art reference

## What does not belong here

- shared worldbuilding that applies broadly across the setting
- act structure and sequencing docs
- active handoff notes
- film-only adaptation materials

Those belong in:
- `story/world/`
- `story/for-agents/`
- `film/`

## Authority

- `character.md` should be the cleanest per-character reference canon.
- `summary.md` should preserve the intended emotional and interpretive read of the character.
- If a character change affects world logic, update the relevant docs in `story/world/` and `story/for-agents/` too.

## Notes

- `official-art/` stores shared official character imagery that is not limited to one character folder.
- Keep naming consistent with the rest of the repo so references remain predictable.
