# Lore Agent Docs

This folder holds fast operational documents used during active story work.

These files are meant to help agents and collaborators load the current state of the project quickly without re-reading the entire canon tree each time.

## Files here

- [canon-primer.md](/C:/Users/leoja/Desktop/Dad%20Games/lore-project/story/for-agents/canon-primer.md): fast-load reference for world, character, object, and reveal essentials
- [current-handoff.md](/C:/Users/leoja/Desktop/Dad%20Games/lore-project/story/for-agents/current-handoff.md): the exact current active story position and immediate locked handling notes
- [story-rules.md](/C:/Users/leoja/Desktop/Dad%20Games/lore-project/story/for-agents/story-rules.md): tone, reveal, character-usage, and collaboration guardrails
- [audit-handoff.md](/C:/Users/leoja/Desktop/Dad%20Games/lore-project/story/for-agents/audit-handoff.md): audit notes and prior synthesis material worth retaining for reference

## What belongs here

- session-start canon shortcuts
- active handoff notes
- operational storytelling rules
- collaboration-facing reference material

## What does not belong here

- the main home for world canon
- the main home for character canon
- adaptation planning
- prose-form canon tracks

Those belong in:
- `story/world/`
- `story/characters/`
- `film/`

## Authority

- These docs are important and may contain canon-sensitive material.
- Even so, this folder is primarily operational, not the main archive of story truth.
- When updating something here that affects canon, make the corresponding change in the deeper source doc as well.

## Workflow

- Update `current-handoff.md` when the active writing position changes.
- Keep `current-handoff.md` aligned with the relevant guide docs in `story/world/story-guides/`.
- Update `story-rules.md` when a storytelling constraint, reveal rule, or collaboration norm becomes locked.
- Update `canon-primer.md` when a repeated fast-load fact changes enough that agents should not rely on older wording.
